## üêõ Issues & Triage
No new issues were opened or closed this week, but there was significant progress on key active issues.

- **New & Active Issues:**
    - **`op-node` Light Client:** Progress was noted on issue [#18317](https://github.com/ethereum-optimism/infra/issues/18317), with confirmation that the `op-node` implementation for following an external source for safe/finalized data is now complete. This feature will enable nodes to mirror external feeds via the `--l2.follow.source` flag.
    - **`op-challenger` Trace Output:** Discussion continued in issue [#18643](https://github.com/ethereum-optimism/infra/issues/18643) regarding the correct trace output when a super root is not found. It was confirmed that the functionality works as expected in the super node trace provider. The issue remains open pending an integration test to ensure alignment between `op-challenger` and `op-program`. It was also decided that `op-supervisor` will not be fixed as it is slated for deletion.
