Discussion continued on investigating upstreaming a Celestia batcher DA size fix [#17872](https://github.com/ethereum-optimism/superchain-ops/issues/17872). This issue is related to [#17796](https://github.com/ethereum-optimism/superchain-ops/issues/17796), which describes a bug where `op-batcher` max frame size is limited by altDA max input size in generic mode, contrary to specifications. sebastianst also commented on [#17796](https://github.com/ethereum-optimism/superchain-ops/issues/17796), suggesting that if a PR is opened with the specified commit from celestiaorg/optimism, it can be merged.
