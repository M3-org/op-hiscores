## üêõ Issues & Triage

### Closed Issues
Key operational and documentation gaps were addressed this week, improving stability and operator experience.

- **Node Synchronization Resolution:** A critical issue where `op-node` v1.16.3 failed to synchronize for Optimism, Base, and Worldchain was resolved. The solution involved upgrading and resyncing `op-geth`, confirming a dependency between the components ([#18510](https://github.com/ethereum-optimism/superchain-ops/issues/18510)).
- **Documentation for Node Operators:** The JSON-RPC API documentation was restructured with separate pages for `op-node` and `op-geth` for better clarity ([#1773](https://github.com/ethereum-optimism/superchain-ops/issues/1773)). Additionally, missing documentation for environment variables corresponding to node flags was restored ([#1874](https://github.com/ethereum-optimism/superchain-ops/issues/1874)).

### New & Active Issues
While past issues were resolved, new and complex node-related problems are under active investigation.

- **New Synchronization Problems:** A significant active issue ([#18581](https://github.com/ethereum-optimism/superchain-ops/issues/18581)) details nodes on Optimism, Base, and Unichain failing to sync with "missing trie node" errors. The problem appears linked to using `op-geth` with snapshots, and maintainers are gathering detailed logs and configurations to diagnose the root cause.
- **Sequencer Recover Mode Bug:** Investigation continues into a possible bug where the `op-node` sequencer incorrectly enters recover mode during normal operation ([#18350](https://github.com/ethereum-optimism/superchain-ops/issues/18350)). Community discussion has linked a potential related PR and proposed a code snippet to prevent temporary errors, indicating active collaborative debugging.
