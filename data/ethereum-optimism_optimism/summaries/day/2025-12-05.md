# ethereum-optimism/optimism Daily Update (Dec 5, 2025)
## OVERVIEW 
Today's development focused on enhancing deployment processes, improving CI/CD security, and expanding testing for core contracts. Key updates include a new `op-deployer` build for artifact publishing and refined CI permissions.

## KEY TECHNICAL DEVELOPMENTS

### Enhanced Deployment and Artifact Management
Improvements were made to the deployment process, specifically by integrating `op-deployer`'s build and compress commands for artifact publishing, ensuring consistency and enabling uploaded artifacts. Additionally, the `FeesDepositor` address and minimum withdrawal amount were updated in genesis configurations.
- feat(op-deployer): use op-deployer build for publish-artifacts ([#18488](https://github.com/ethereum-optimism/optimism/pull/18488))
- chore: update fees depositor address and min withdrawal amount on genesis ([#18490](https://github.com/ethereum-optimism/optimism/pull/18490))

### CI/CD Security and Testing Improvements
The CI workflows were refined to reduce permissions for fork builds and fix tag releases, separating release tag pushes from branch and PR builds for enhanced security. New tests were also added for `OPContractsManagerV2.deploy`.
- ci: reduce permissions of fork builds & fix tag releases ([#18501](https://github.com/ethereum-optimism/optimism/pull/18501))
- test: add tests for OPCMv2.deploy ([#18486](https://github.com/ethereum-optimism/optimism/pull/18486))

### Conductor Healthcheck Expansion
The `op-conductor` now includes first-class support for Kona health probes, alongside the existing rollup-boost healthcheck, addressing the need for a dedicated health endpoint for Kona.
- Add kona-healthcheck support to conductor ([#18453](https://github.com/ethereum-optimism/optimism/pull/18453))
