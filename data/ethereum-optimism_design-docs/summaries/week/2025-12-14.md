## üêõ Issues & Triage
Issue management this week saw the closure of two notable items and the report of a new bug.

- **Closed Issues:**
    - **Blob Priority Fee Tracking ([#18102](https://github.com/ethereum-optimism/design-docs/issues/18102)):** This issue was closed following the confirmation that the blob tip oracle is now deployed to production. The oracle's data suggests the system is currently overpaying for blob transactions, indicating that work is needed to integrate the oracle's fee suggestions.
    - **Sequencer Recovery Mode Bug ([#18350](https://github.com/ethereum-optimism/design-docs/issues/18350)):** This issue, which described a potential drift problem, was closed after the behavior could not be reproduced. However, related refactoring of the L1 origin code in `op-node` was implemented as a preventative measure.

- **New & Active Issues:**
    - **Artifact Extraction Path Discrepancy ([#18663](https://github.com/ethereum-optimism/design-docs/issues/18663)):** A new bug was reported in `op-deployer`. The `ExtractFromFile()` function incorrectly expects an `out/` directory instead of `forge-artifacts/`, which breaks the use of self-hosted `.tzst` artifacts.
