## üêõ Issues & Triage
Issue management this week involved resolving older sync and documentation issues while a major new synchronization problem took center stage.

- **Closed Issues:**
    - **Node Synchronization:** An issue where an EL node failed to sync after an `op-node` update was resolved by upgrading and resyncing `op-geth` ([#18510](https://github.com/ethereum-optimism/op-geth/issues/18510)).
    - **Documentation:** Two issues were closed to improve developer experience: one for updating the JSON-RPC API documentation ([#1773](https://github.com/ethereum-optimism/op-geth/issues/1773)) and another for making environment variables available in the docs ([#1874](https://github.com/ethereum-optimism/op-geth/issues/1874)).

- **New & Active Issues:**
    - **Widespread Sync Failures:** A critical new issue ([#18581](https://github.com/ethereum-optimism/op-geth/issues/18581)) emerged, with multiple users on Base, Optimism, and Unichain reporting that nodes running `op-geth v1.101603.5 jovian` are failing to synchronize. Users are encountering "missing trie node" and "Failed to build payload" errors, particularly when using public snapshots. This appears to be a significant blocker for node operators.
    - **Planned Refactoring:** Discussion continues on removing the `isPectra` flag from the batcher ([#18479](https://github.com/ethereum-optimism/op-geth/issues/18479)), with a contributor expressing interest in taking on the task.
