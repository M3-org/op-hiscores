- **New & Active Issues:**

    - An important architectural discussion continues regarding how to keep the `op-batcher` synchronized with the `op-node` rollup configuration ([#18178](https://github.com/ethereum-optimism/ecosystem/issues/18178)). The current implementation, which fetches the config only at startup, is considered brittle. Proposed solutions include making the dependency explicit by having the operator provide the config, or using endpoint changes to trigger a configuration reload.
