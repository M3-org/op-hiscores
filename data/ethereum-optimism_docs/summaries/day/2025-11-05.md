## ACTIVE ISSUES

### op-batcher: Stay in Sync with op-node Rollup Config ([#18178](https://github.com/ethereum-optimism/docs/issues/18178))
Discussion revolves around the `op-batcher`'s current behavior of fetching the rollup configuration only once at startup from the `op-node` via RPC, and never updating it thereafter. This approach is considered brittle, as changes to the rollup config are not dynamically reflected. Suggestions include making the operator explicitly provide the rollup config to the batcher to make the dependency clearer, or implementing a mechanism where the batcher reloads the config when the active endpoint provider signals a change. The trade-off for the explicit provision approach is that a new batcher release would be needed for each Superchain Registry config change, which is not currently the case.
