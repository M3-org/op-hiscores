On November 13, 2025, Optimism significantly enhanced its supply chain security by implementing SLSA Build Level 3 attestations for Docker images across `op-geth`, `ecosystem`, and `infra` repositories, alongside critical updates to `op-node` for improved synchronization and testing infrastructure. Documentation for L2 contract management and chain operators also saw substantial improvements, while discussions continued on adapting the `print_staging_report` script for Dispute Game v2 compatibility and adding support for embedded custom prestates to Kona.

## ‚úÖ Completed Work

### Supply Chain Security & CI/CD Enhancements
- `op-geth`: Added GitHub Actions workflow to build Docker images with SLSA Build Level 3 in-toto provenance attestations signed with Sigstore, improving supply chain security ([#720](https://github.com/ethereum-optimism/op-geth/pull/720)).
- `ecosystem`: Added GitHub Actions workflows for building and attesting ecosystem Docker images ([#970](https://github.com/ethereum-optimism/ecosystem/pull/970)).
- `infra`: Introduced new GitHub Actions workflows to automate building, attestation, and signing of Docker images for all infrastructure components, enhancing security and reliability ([#510](https://github.com/ethereum-optimism/infra/pull/510)).

### Node Synchronization & Performance
- `optimism`: `op-node` was updated to perform ELSync gap filling and deprecate RR sync, improving node synchronization ([#17751](https://github.com/ethereum-optimism/optimism/pull/17751)).
- `optimism`: Enabled P2P for Virtual Nodes in `op-supernode` ([#18213](https://github.com/ethereum-optimism/optimism/pull/18213)).
- `op-geth`: Introduced `RollupNetrestrictTxPoolGossipFlag` to control transaction pool gossip behavior ([#706](https://github.com/ethereum-optimism/op-geth/pull/706)).
- `op-geth`: The downloader now skips the `blobGasUsed` check for Optimism chains on Jovian ([#723](https://github.com/ethereum-optimism/op-geth/pull/723)).

### Documentation & Design Improvements
- `design-docs`: Added a new design document outlining deterministic, hard fork-driven upgrades of L2 contracts, improving the upgrade process for protocol developers ([#351](https://github.com/ethereum-optimism/design-docs/pull/351)).
- `docs`: Created a new guide on how to set the DA footprint ([#1858](https://github.com/ethereum-optimism/docs/pull/1858)).
- `docs`: Improved existing DA Footprint documentation ([#1864](https://github.com/ethereum-optimism/docs/pull/1864)).
- `docs`: Updated Flashblocks RPC documentation to cover all affected RPC methods ([#1863](https://github.com/ethereum-optimism/docs/pull/1863)).
- `docs`: Attached information about the flashblocks websocket endpoints ([#1862](https://github.com/ethereum-optimism/docs/pull/1862)).
- `optimism`: Added a rule to Graphite documentation to prevent flagging correct `vm.expectRevert` patterns with low-level calls ([#18272](https://github.com/ethereum-optimism/optimism/pull/18272)).

### Testing & Infrastructure Updates
- `optimism`: Improved RLPReader test coverage and organization ([#18232](https://github.com/ethereum-optimism/optimism/pull/18232)).
- `optimism`: Updated `op-acceptor` to v3.6.6 for acceptance tests ([#18266](https://github.com/ethereum-optimism/optimism/pull/18266)).
