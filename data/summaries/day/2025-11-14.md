On Nov 14, 2025, the Optimism project made significant strides in enhancing `op-program` and Kona client compatibility, particularly around block header lookups and websocket support, while also strengthening CI/CD processes and dispute game tooling. Core contract updates for OPCMv2 and improvements to the dispute monitoring system underscore a focus on foundational infrastructure and security.

## ‚úÖ Completed Work

### Core Protocol & Client Compatibility
*   **`op-program` Hinting for Fast Block Header Lookup**: A hint was added to `op-program` to facilitate fast block header oracle lookups, aligning with `kona-client`'s direct storage slot lookup approach. ([ethereum-optimism/op-geth#17416](https://github.com/ethereum-optimism/op-geth/issues/17416), [ethereum-optimism/ecosystem#17416](https://github.com/ethereum-optimism/ecosystem/issues/17416), [ethereum-optimism/docs#17416](https://github.com/ethereum-optimism/docs/issues/17416), [ethereum-optimism/superchain-registry#17416](https://github.com/ethereum-optimism/superchain-registry/issues/17416), [ethereum-optimism/specs#17416](https://github.com/ethereum-optimism/specs/issues/17416), [ethereum-optimism/superchain-ops#17416](https://github.com/ethereum-optimism/superchain-ops/issues/17416), [ethereum-optimism/optimism#17416](https://github.com/ethereum-optimism/optimism/issues/17416))
*   **Kona Compatibility with Websocket URLs**: Kona was updated to resolve compatibility issues with websocket URLs in acceptance tests, addressing previous incompatibilities with L2 EL connections. ([ethereum-optimism/op-geth#18203](https://github.com/ethereum-optimism/op-geth/issues/18203), [ethereum-optimism/ecosystem#18203](https://github.com/ethereum-optimism/ecosystem/issues/18203), [ethereum-optimism/docs#18203](https://github.com/ethereum-optimism/docs/issues/18203), [ethereum-optimism/superchain-registry#18203](https://github.com/ethereum-optimism/superchain-registry/issues/18203), [ethereum-optimism/specs#18203](https://github.com/ethereum-optimism/specs/issues/18203), [ethereum-optimism/superchain-ops#18203](https://github.com/ethereum-optimism/superchain-ops/issues/18203), [ethereum-optimism/optimism#18203](https://github.com/ethereum-optimism/optimism/issues/18203))
*   **Kona Prestate Support**: Tracking issue for adding support for embedded custom prestates to Kona was closed, with a fix under review. ([ethereum-optimism/optimism#18271](https://github.com/ethereum-optimism/optimism/issues/18271))

### Infrastructure & CI/CD Enhancements
*   **CI/CD for Docker Image Builds**: New GitHub Actions workflows were introduced to build and attest Docker images for `ecosystem-ponder-interop`, `ecosystem-autorelayer-interop`, and `ecosystem-sponsored-sender`. ([ethereum-optimism/ecosystem#970](https://github.com/ethereum-optimism/ecosystem/pull/970))
*   **Dispute Game Report Tooling**: The `print-staging-report` command was updated to correctly fetch permissioned dispute game parameters, improving compatibility with newer dispute game configurations. ([ethereum-optimism/superchain-registry#1150](https://github.com/ethereum-optimism/superchain-registry/pull/1150), [ethereum-optimism/superchain-registry#1148](https://github.com/ethereum-optimism/superchain-registry/issues/1148))
*   **Standard Prestate Updates**: `kona-client/v1.2.4` was added to the standard prestates for up-to-date configurations. ([ethereum-optimism/superchain-registry#1154](https://github.com/ethereum-optimism/superchain-registry/pull/1154))
*   **Core Contracts and Dispute System Enhancements**: Updates to `AnchorStateRegistry` for OPCMv2 and refactoring of `op-dispute-mon` for clarity. ([ethereum-optimism/optimism#18244](https://github.com/ethereum-optimism/optimism/pull/18244), [ethereum-optimism/optimism#18278](https://github.com/ethereum-optimism/optimism/pull/18278))
*   **Deployment and Testing Infrastructure Improvements**: Critical bug fix for `SuperchainConfigProxy` and configurable attempts for "EL NotAdvanced" scenarios in `op-devstack`. ([ethereum-optimism/optimism#18293](https://github.com/ethereum-optimism/optimism/pull/18293), [ethereum-optimism/optimism#18286](https://github.com/ethereum-optimism/optimism/pull/18286))
*   **CI/CD Workflow Streamlining**: CI pipeline updates to reinstate and properly wire the `initialize` job and add `--initialize` flag requirement for subsequent jobs. ([ethereum-optimism/optimism#18284](https://github.com/ethereum-optimism/optimism/pull/18284), [ethereum-optimism/optimism#18285](https://github.com/ethereum-optimism/optimism/pull/18285))

### Documentation & U17 Task Management
*   **Documentation Structure Refinement**: The issue concerning the implementation of a new navigation structure for the documentation site has been closed. ([ethereum-optimism/ecosystem#926](https://github.com/ethereum-optimism/ecosystem/issues/926))
*   **`op-geth` Rewind Tutorial**: A new tutorial was added to guide users on how to rewind `op-geth`. ([ethereum-optimism/docs#1866](https://github.com/ethereum-optimism/docs/pull/1866))
*   **Notices and Release Versioning**: Updates were made to `fusaka-notice.mdx` and `upgrade-17.mdx` including rust stack releases and general versioning. ([ethereum-optimism/docs#1868](https://github.com/ethereum-optimism/docs/pull/1868), [ethereum-optimism/docs#1867](https://github.com/ethereum-optimism/docs/pull/1867))
*   **U17 Task Management and Documentation**: Corrected READMEs for U17 sign commands and updated U17 Sepolia tasks to "executed". ([ethereum-optimism/superchain-ops#1292](https://github.com/ethereum-optimism/superchain-ops/pull/1292), [ethereum-optimism/superchain-ops#1297](https://github.com/ethereum-optimism/superchain-ops/pull/1297), [ethereum-optimism/superchain-ops#1293](https://github.com/ethereum-optimism/superchain-ops/pull/1293))
