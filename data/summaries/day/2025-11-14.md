On November 14, 2025, Optimism's development focused heavily on enhancing compatibility and robustness across the Superchain, particularly with the Kona client and `op-program`. Significant progress was made in refining CI/CD processes for Docker image builds and updating core contracts for OPCMv2, alongside crucial documentation updates and dispute game parameter extraction improvements. The team also initiated new work on L2CM specifications and ZK Proofs, indicating a forward-looking approach to scaling and security.

### Superchain Interoperability & Kona Client Compatibility
- **`op-program` Hinting for Fast Block Header Lookup**: A hint was added to `op-program` to provide key information for fast block header oracle lookups, benefiting `kona-client`'s direct storage slot lookup. ([ethereum-optimism/op-geth#17416](https://github.com/ethereum-optimism/op-geth/issues/17416), [ethereum-optimism/ecosystem#17416](https://github.com/ethereum-optimism/ecosystem/issues/17416), [ethereum-optimism/docs#17416](https://github.com/ethereum-optimism/docs/issues/17416), [ethereum-optimism/superchain-registry#17416](https://github.com/ethereum-optimism/superchain-registry/issues/17416), [ethereum-optimism/specs#17416](https://github.com/ethereum-optimism/specs/issues/17416), [ethereum-optimism/superchain-ops#17416](https://github.com/ethereum-optimism/superchain-ops/issues/17416), [ethereum-optimism/optimism#17416](https://github.com/ethereum-optimism/optimism/issues/17416))
- **Kona Compatibility with Acceptance Tests**: Kona was updated to resolve compatibility issues with websocket URLs in acceptance tests, ensuring smoother integration with L2 EL connections. ([ethereum-optimism/op-geth#18203](https://github.com/ethereum-optimism/op-geth/issues/18203), [ethereum-optimism/ecosystem#18203](https://github.com/ethereum-optimism/ecosystem/issues/18203), [ethereum-optimism/docs#18203](https://github.com/ethereum-optimism/docs/issues/18203), [ethereum-optimism/superchain-registry#18203](https://github.com/ethereum-optimism/superchain-registry/issues/18203), [ethereum-optimism/specs#18203](https://github.com/ethereum-optimism/specs/issues/18203), [ethereum-optimism/superchain-ops#18203](https://github.com/ethereum-optimism/superchain-ops/issues/18203), [ethereum-optimism/optimism#18203](https://github.com/ethereum-optimism/optimism/issues/18203))
- **Kona Prestate Support**: Support for embedded custom prestates was added to Kona. ([ethereum-optimism/optimism#18271](https://github.com/ethereum-optimism/optimism/issues/18271))

### CI/CD & Core Infrastructure Improvements
- **CI/CD Enhancements for Docker Image Builds**: New GitHub Actions workflows were introduced to build and attest Docker images for `ecosystem-ponder-interop`, `ecosystem-autorelayer-interop`, and `ecosystem-sponsored-sender`. ([ethereum-optimism/ecosystem#970](https://github.com/ethereum-optimism/ecosystem/pull/970))
- **Core Contracts and Deployment Enhancements**: Updates were made to `AnchorStateRegistry` for OPCMv2 and a bug fix was implemented for `SuperchainConfigProxy` in deployer integration tests. ([ethereum-optimism/optimism#18244](https://github.com/ethereum-optimism/optimism/pull/18244), [ethereum-optimism/optimism#18293](https://github.com/ethereum-optimism/optimism/pull/18293))
- **Dispute Game Parameter Extraction**: The `print-staging-report` command was updated to correctly fetch permissioned dispute game parameters from game arguments, improving accuracy for dispute games configured with game args. ([ethereum-optimism/superchain-registry#1150](https://github.com/ethereum-optimism/superchain-registry/pull/1150), [ethereum-optimism/superchain-registry#1148](https://github.com/ethereum-optimism/superchain-registry/issues/1148))
- **Dispute Monitoring Improvements**: The dispute monitoring system was refactored, renaming `L2BlockNumber` to `L2SequenceNumber`, and acceptance tests were enhanced with configurable attempts for "EL Not Advanced" scenarios. ([ethereum-optimism/optimism#18278](https://github.com/ethereum-optimism/optimism/pull/18278), [ethereum-optimism/optimism#18286](https://github.com/ethereum-optimism/optimism/pull/18286))
- **CI Release Workflow & Dependency Updates**: The CI release workflow's `initialize` job was reinstated and properly wired, and the `op-geth` dependency was upgraded across the project. ([ethereum-optimism/optimism#18285](https://github.com/ethereum-optimism/optimism/pull/18285), [ethereum-optimism/optimism#18284](https://github.com/ethereum-optimism/optimism/pull/18284), [ethereum-optimism/optimism#18280](https://github.com/ethereum-optimism/optimism/pull/18280))

### Documentation & Operator Tooling
- **Documentation Structure Update**: The issue concerning the implementation of a new navigation structure for the documentation site has been closed. ([ethereum-optimism/ecosystem#926](https://github.com/ethereum-optimism/ecosystem/issues/926))
- **Notices and Version Updates**: Updates were made to notices concerning rust stack releases and general versioning. ([ethereum-optimism/docs#1868](https://github.com/ethereum-optimism/docs/pull/1868), [ethereum-optimism/docs#1867](https://github.com/ethereum-optimism/docs/pull/1867))
- **New Operator Tutorial**: A new tutorial was added to guide users on how to rewind `op-geth`. ([ethereum-optimism/docs#1866](https://github.com/ethereum-optimism/docs/pull/1866))
- **U17 Task Management and Documentation**: README files for U17 sign commands were corrected, and the status of U17 Sepolia tasks was updated to "executed." ([ethereum-optimism/superchain-ops#1292](https://github.com/ethereum-optimism/superchain-ops/pull/1292), [ethereum-optimism/superchain-ops#1297](https://github.com/ethereum-optimism/superchain-ops/pull/1297), [ethereum-optimism/superchain-ops#1293](https://github.com/ethereum-optimism/superchain-ops/pull/1293))
