## ‚úÖ Completed Work
### Infrastructure & Protocol Enhancements
- **Jovian Upgrade (U17) Preparations and Fixes**: Critical bug fixes related to gas limits for the U17 upgrade on betanets and sepolia-dev-0 were implemented, alongside necessary task and configuration additions. ([ethereum-optimism/superchain-ops#1273](https://github.com/ethereum-optimism/superchain-ops/pull/1273), [ethereum-optimism/superchain-ops#1280](https://github.com/ethereum-optimism/superchain-ops/pull/1280))
- **Rollup Configuration and Finality Enhancements**: New optional configuration fields were added to the rollup config for default finality lookback, making finality behavior configurable, and fork override flags were modularized for forwards compatibility. ([ethereum-optimism/optimism#18071](https://github.com/ethereum-optimism/optimism/pull/18071), [ethereum-optimism/optimism#18135](https://github.com/ethereum-optimism/optimism/pull/18135))
- **Contract Bedrock Improvements and SaferSafes Audit Fixes**: The `ScheduledTransaction` struct in `TimelockGuard` was extended to include `txHash` and `nonce`, and fixes from the SaferSafes audit were implemented to address silent failures in `execTransactionFromModule` calls. ([ethereum-optimism/optimism#18097](https://github.com/ethereum-optimism/optimism/pull/18097), [ethereum-optimism/optimism#18147](https://github.com/ethereum-optimism/optimism/pull/18147))
- **Deployment Tooling and Infrastructure Updates**: The `justfile` in `op-deployer` was updated to utilize current CLI flags, debug information was added to `op-devstack` for L2CL peers connection failures, `op-geth` was updated to incorporate Fusaka and Jovian timestamps, and `op-deployer` now supports the `op-contracts/v5.0.0-rc.2` upgrade command. ([ethereum-optimism/optimism#18143](https://github.com/ethereum-optimism/optimism/pull/18143), [ethereum-optimism/optimism#18149](https://github.com/ethereum-optimism/optimism/pull/18149), [ethereum-optimism/optimism#18131](https://github.com/ethereum-optimism/optimism/pull/18131), [ethereum-optimism/optimism#18154](https://github.com/ethereum-optimism/optimism/pull/18154))
- **Core Protocol Definition Refactoring**: The `predeploys` package was moved from `op-service` to `op-core` as it is a core protocol definition. ([ethereum-optimism/optimism#18145](https://github.com/ethereum-optimism/optimism/pull/18145))
- **Validation and Deployment Updates**: `op-program v1.8.0-rc.4` prestate was added to `standard-prestates.toml` for validation, and MIPS and OPCM deployments were updated in `standard-versions-mainnet.toml` and `standard-versions-sepolia.toml`. ([ethereum-optimism/superchain-registry#1142](https://github.com/ethereum-optimism/superchain-registry/pull/1142), [ethereum-optimism/superchain-registry#1141](https://github.com/ethereum-optimism/superchain-registry/pull/1141))
