On Dec 3, 2025, the Optimism project made significant strides in enhancing the `op-challenger`'s support for ZK game types, with numerous related issues closed across multiple repositories, indicating a major push towards integrating ZK proof capabilities. Concurrently, critical bug fixes were implemented for `op-node` and `op-batcher` stability, alongside ongoing discussions regarding an `op-node` EL sync issue affecting Base Mainnet nodes and the need for comprehensive acceptance tests for new EIPs.

## üö® Needs Attention
- **Urgent Discussions**:
    - `op-node` EL Sync Mode Issue ([#18468](https://github.com/ethereum-optimism/op-analytics/issues/18468)): Multiple users are experiencing `op-node` getting stuck in EL sync mode after a restart, blocking Base Mainnet nodes and requiring immediate investigation.
    - Acceptance Tests for New EIPs ([#18221](https://github.com/ethereum-optimism/op-analytics/issues/18221)): Discussion is needed on adding acceptance tests to ensure features not included in `fusaka-on-l2` are properly disabled on OP chains.
    - Unused OP Constants ([#13440](https://github.com/ethereum-optimism/op-analytics/issues/13440)): Confirmation is needed on whether this issue is resolved and can be closed.

## ‚úÖ Completed Work
- **ZK Challenger and Proof Generation Advancements**:
    - The decision was finalized to use the succinct proposer for ZK proof generation across `op-analytics` ([#18319](https://github.com/ethereum-optimism/op-analytics/issues/18319)), `infra` ([#18319](https://github.com/ethereum-optimism/infra/issues/18319)), `superchain-registry` ([#18319](https://github.com/ethereum-optimism/superchain-registry/issues/18319)), `superchain-ops` ([#18319](https://github.com/ethereum-optimism/superchain-ops/issues/18319)), `docs` ([#18319](https://github.com/ethereum-optimism/docs/issues/18319)), and `optimism` ([#18319](https://github.com/ethereum-optimism/optimism/issues/18319)).
    - Support for challenging ZK game types in `op-challenger` was implemented and closed across `op-analytics` ([#18321](https://github.com/ethereum-optimism/op-analytics/issues/18321)), `infra` ([#18321](https://github.com/ethereum-optimism/infra/issues/18321)), `superchain-registry` ([#18321](https://github.com/ethereum-optimism/superchain-registry/issues/18321)), `superchain-ops` ([#18321](https://github.com/ethereum-optimism/superchain-ops/issues/18321)), `docs` ([#18321](https://github.com/ethereum-optimism/docs/issues/18321)), and `optimism` ([#18321](https://github.com/ethereum-optimism/optimism/issues/18321)).
    - The `op-challenger` was enhanced to challenge unsafe ZK proposals ([#18467](https://github.com/ethereum-optimism/optimism/pull/18467)), reclaim bonds ([#18465](https://github.com/ethereum-optimism/optimism/pull/18465)), and resolve optimistic ZK games ([#18464](https://github.com/ethereum-optimism/optimism/pull/18464)).
- **Challenger Configuration and Robustness Improvements**:
    - Multiple issues related to `op-challenger` configuration and robustness were closed, including improvements to test coverage ([#18427](https://github.com/ethereum-optimism/op-analytics/issues/18427)), reading from the DGF registry ([#18412](https://github.com/ethereum-optimism/op-analytics/issues/18412)), fetching challenger bonds ([#18413](https://github.com/ethereum-optimism/op-analytics/issues/18413)), reloading bond values ([#18414](https://github.com/ethereum-optimism/op-analytics/issues/18414)), and supporting challenger address changes ([#18415](https://github.com/ethereum-optimism/op-analytics/issues/18415)) across `op-analytics`, `infra`, `superchain-registry`, `superchain-ops`, and `docs`.
- **Core Infrastructure Stability and Security**:
    - The `replica-healthcheck` Docker image build process was aligned to use the repository root as the build context in `infra` ([#521](https://github.com/ethereum-optimism/infra/pull/521)).
    - A new script was added to deploy safer safes in `optimism` ([#18448](https://github.com/ethereum-optimism/optimism/pull/18448)).
    - Critical bugs were fixed in `optimism` for the L1 origin selector getting stuck on reorgs in `op-node` ([#18233](https://github.com/ethereum-optimism/optimism/pull/18233)) and `MaxChannelDuration` tracking in `op-batcher` ([#18327](https://github.com/ethereum-optimism/optimism/pull/18327)).
- **Data Analytics and Ecosystem Enhancements**:
    - DEX Trades Logic in `op-analytics` was updated to include Solana in the query pipeline and refine filtering criteria ([#1770](https://github.com/ethereum-optimism/op-analytics/pull/1770)).
    - An automated code generation update was performed in `superchain-registry` ([#1173](https://github.com/ethereum-optimism/superchain-registry/pull/1173)).
    - Worldchain integration and configuration updates were made in `superchain-ops`, including setting the L2PAO key handback as executed on Sepolia ([#1315](https://github.com/ethereum-optimism/superchain-ops/pull/1315)) and updating test configurations for "soneium" ([#1309](https://github.com/ethereum-optimism/superchain-ops/pull/1309)).
