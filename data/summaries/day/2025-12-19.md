## ‚úÖ Completed Work

### Protocol Simplification & Core Infrastructure
- **Kona Node Actor Design Simplification**: A proposal was introduced to simplify the Kona node actor design, aiming to decrease complexity and increase testability by consolidating and deleting unnecessary code without altering functionality. [ethereum-optimism/design-docs#358](https://github.com/ethereum-optimism/design-docs/pull/358)
- **Blob Gas Price Oracle Module**: A new `BlobGasPriceOracle` module was introduced in `op-service` to track and cache blob prices from new blocks and headers, similar to the existing `GasPriceOracle`. [ethereum-optimism/optimism#18386](https://github.com/ethereum-optimism/optimism/pull/18386)

### Deployment & Developer Experience Enhancements
- **OPCM v2 Support in `op-deployer`**: `op-deployer` received a major upgrade with support for OP Contracts Manager v2 (OPCM v2), enabling more robust chain upgrades including version detection and handling of `UpgradeInputV2`. [ethereum-optimism/optimism#18593](https://github.com/ethereum-optimism/optimism/pull/18593)
- **`op-deployer` Max Code Size Limit Ignored**: `op-deployer` was updated to ignore max-code-size limits during deployment, improving the developer experience for devnet setups. [ethereum-optimism/optimism#18642](https://github.com/ethereum-optimism/optimism/pull/18642)
- **Dune Pulls Modification**: A minor modification was made to the Dune pulls, affecting the `.circleci/config.yml` and `other_chains_tracking/get_qualified_txs.ipynb` files. [ethereum-optimism/op-analytics#1777](https://github.com/ethereum-optimism/op-analytics/pull/1777)
- **World Chain Sepolia Token Integration**: A new token, AMIE, was added to the World Chain Sepolia testnet, including its details and logo, expanding supported tokens. [ethereum-optimism/ethereum-optimism.github.io#1243](https://github.com/ethereum-optimism/ethereum-optimism.github.io/pull/1243)

### Sync Tester & Devstack Improvements
- **`op-sync-tester` `engine_exchangeCapabilities` Support**: The `op-sync-tester` now supports `engine_exchangeCapabilities`, a requirement for Kona integration. [ethereum-optimism/optimism#18628](https://github.com/ethereum-optimism/optimism/pull/18628)
- **`op-devstack` Kona Light CL Support**: `op-devstack` was updated to support Kona light CL by allowing the `KONA_NODE_L2_FOLLOW_SOURCE` environment variable. [ethereum-optimism/optimism#18627](https://github.com/ethereum-optimism/optimism/pull/18627)
