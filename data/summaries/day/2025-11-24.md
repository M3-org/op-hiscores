## ‚úÖ Completed Work
### Core Protocol Enhancements
- **Custom Gas Token Integration**: A major feature was introduced to allow OP Stack chains to utilize assets other than ETH as their native fee currency, involving extensive modifications across acceptance tests, chain operations, deployer configurations, and bedrock contracts. ([ethereum-optimism/optimism#18076](https://github.com/ethereum-optimism/optimism/pull/18076))
- **Safe Contracts Updates**: All Safe directory changes from the `develop` branch were cherry-picked to the `op-safe-contracts/v1.0.0` branch, ensuring the v1.0.0 branch reflects the latest improvements. ([ethereum-optimism/optimism#18371](https://github.com/ethereum-optimism/optimism/pull/18371))

### Testing and CI/CD Improvements
- **`op-reth` Integration Confirmed**: The re-addition of `op-reth` to the Jovian kt-devnet for `op-acceptance-test` has been confirmed and closed, indicating `op-reth` is now running acceptance tests within `op-rs/kona` alongside various configurations. ([ethereum-optimism/op-geth#17475](https://github.com/ethereum-optimism/op-geth/issues/17475), [ethereum-optimism/infra#17475](https://github.com/ethereum-optimism/infra/issues/17475), [ethereum-optimism/docs#17475](https://github.com/ethereum-optimism/docs/issues/17475), [ethereum-optimism/optimism#17475](https://github.com/ethereum-optimism/optimism/issues/17475))
- **CI/CD Pipeline Enhancements**: Improvements were made to the CI/CD pipeline, including updating heavy fuzz tests to incorporate dev feature matrices and allocating more resources to `op-e2e-cannon-tests` to prevent resource limits from killing jobs. ([ethereum-optimism/optimism#18368](https://github.com/ethereum-optimism/optimism/pull/18368), [ethereum-optimism/optimism#18359](https://github.com/ethereum-optimism/optimism/pull/18359), [ethereum-optimism/optimism#18367](https://github.com/ethereum-optimism/optimism/pull/18367))

### Documentation and Infrastructure Refactoring
- **Fault Proof System Documentation**: An issue related to updating OP Labs runbooks for the procedure to switch to Kona + Cannon as a fault proof was closed, indicating progress in documenting the fault proof system. ([ethereum-optimism/optimism#17287](https://github.com/ethereum-optimism/optimism/issues/17287))
- **Code Refactoring**: Unnecessary reassignments were removed in a refactoring effort. ([ethereum-optimism/infra#518](https://github.com/ethereum-optimism/infra/pull/518))
