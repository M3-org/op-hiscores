## ACTIVE ISSUES

### op-node Synchronization Issues
Discussions continued regarding the inability of op-node v1.16.3 to synchronize for Optimism, Base, and Worldchain deployments. The latest comment provided specific docker image versions for op-geth and op-node being used.
- [#18510](https://github.com/ethereum-optimism/Retro-Funding/issues/18510): Users are experiencing synchronization failures after upgrading op-node to v1.16.3. The latest comment from J1a-wei specifies the exact `op-geth` (`v1.101603.4`) and `op-node` (`v1.16.3`) docker image versions being used.

### op-acceptance-tests and Flashblocks Support
The issue concerning `sysgo`'s lack of awareness of flashblocks/rbuilders in `op-acceptance-tests` was revisited. A recent comment from geoknee questioned if the task is complete.
- [#16450](https://github.com/ethereum-optimism/Retro-Funding/issues/16450): The `op-acceptance-tests` currently cannot run flashblocks-related tests with the `sysgo` orchestrator. geoknee asked for an update on whether this task is completed.

### New op-spammer Component for Devnets
Discussion on creating a new `op-spammer` component for devnet deployments included a suggestion to look at existing solutions for prior art.
- [#18482](https://github.com/ethereum-optimism/Retro-Funding/issues/18482): A new `op-spammer` component is proposed to help catch bugs on devnets by periodically sending user transactions. geoknee suggested looking at `https://github.com/flashbots/contender` as prior art.
