## üêõ Issues & Triage
Issue management this month saw the resolution of a flaky test and active discussions on improving core infrastructure components.

- **Closed Issues:**
  - A long-standing flaky test, `TestL2CLAheadOfSupervisor`, was closed after no longer appearing in frequent failure reports, improving CI stability ([#17548](https://github.com/ethereum-optimism/Retro-Funding/issues/17548)).
  - An issue tracking an audit for OPCM changes was closed, as its scope is now covered by a broader issue in the main `optimism` repository ([#17371](https://github.com/ethereum-optimism/Retro-Funding/issues/17371)).

- **New & Active Issues:**
  - **`op-batcher` Enhancements:** Several active discussions centered on improving the `op-batcher`:
    - **Configuration Sync ([#18178](https://github.com/ethereum-optimism/Retro-Funding/issues/18178)):** A key discussion is underway regarding how the `op-batcher` should handle configuration changes from `op-node`, weighing the trade-offs between explicit configuration updates and a dynamic reloading mechanism.
    - **Blob Fee Tracking ([#18102](https://github.com/ethereum-optimism/Retro-Funding/issues/18102)):** The team is exploring methods to improve blob priority fee tracking, considering both a simple decay-based model and a more robust dedicated tracker.
    - **Spec Alignment ([#17796](https://github.com/ethereum-optimism/Retro-Funding/issues/17796)):** Following a bug fix, a discussion emerged about the need to update the Optimism specifications to accurately reflect the current implementation of max frame size limits.
  - **`proxyd` Reliability ([#501](https://github.com/ethereum-optimism/Retro-Funding/issues/501)):** An ongoing conversation aims to solve the "backwards time travel" issue that occurs when `proxyd` backends are out of sync. Proposed solutions include multicall strategies and various sticky session implementations.
  - **Tooling & Compatibility:**
    - It was confirmed that a fix for an incorrect keccak hash issue affecting Cannon was included in Go 1.25.4 ([#17060](https://github.com/ethereum-optimism/Retro-Funding/issues/17060)).
    - Work is in progress to add Jovian hard fork support to the `op-sync-tester` ([#18126](https://github.com/ethereum-optimism/Retro-Funding/issues/18126)).
